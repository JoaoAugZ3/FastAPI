git init: Inicializa um novo repositório Git em um diretório vazio ou converte um diretório existente em um repositório Git.

git clone URL: Clona um repositório Git remoto para o seu computador, criando uma cópia local do projeto.

git add ARQUIVO: Adiciona um arquivo ao índice (staging area), preparando-o para um commit.

git commit -m "MENSAGEM": Realiza um commit das alterações no índice, registrando as mudanças no histórico do repositório com uma mensagem descritiva.

git pull: Atualiza o repositório local com as alterações do repositório remoto. Ele realiza um git fetch seguido de um git merge.

git push: Envia os commits locais para o repositório remoto, atualizando-o com as suas alterações.

git branch: Lista os branches existentes no repositório e destaca o branch atual.

git checkout BRANCH: Alterna para um branch específico.

git merge BRANCH: Mescla as alterações de um branch em outro. É usado para combinar o trabalho de diferentes branches.

git stash: Salva as mudanças locais em um estado temporário, permitindo que você as remova temporariamente do seu branch para trabalhar em outro lugar.

git log: Exibe um histórico de commits no branch atual.

git status: Exibe o estado atual do seu diretório de trabalho, mostrando os arquivos modificados, adicionados ou excluídos.

git remote -v: Exibe informações sobre os repositórios remotos configurados para o seu projeto.

git fetch: Busca as alterações dos repositórios remotos sem mesclá-las com o branch local.

git reset: Permite desfazer commits, redefinir o índice e o diretório de trabalho para um estado anterior.

git rebase: Reorganiza a história de commits, aplicando commits de um branch em outro.

git tag: Permite criar, listar e gerenciar tags (rótulos) para marcar versões específicas do seu código.
